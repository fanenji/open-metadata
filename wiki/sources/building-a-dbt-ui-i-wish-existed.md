# Building a dbt-UI I Wish Existed

An article by Remis Haroon discussing the creation of **dbt-Workbench**, a self-hosted, open-source UI designed to reduce friction in dbt workflows, particularly for engineers working in local, on-premise, or air-gapped environments.

The author argues that while dbt Cloud is excellent for cloud-native teams, it introduces significant friction in restricted networks. The proposed solution, dbt-Workbench, uses existing dbt artifacts (`manifest.json`, `run_results.json`, `catalog.json`) to provide lineage visualization, model browsing, and SQL inspection without requiring a remote service or a running database.
