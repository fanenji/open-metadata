# How to work with the Incident Manager | Official Documentation

This source is the official procedural guide for the Incident Manager feature in OpenMetadata v1.12.x. It documents the four-step workflow for managing data quality incidents: using the Incident Dashboard, changing incident status, resolving and closing incidents, and viewing incident activities. The guide covers available filters (Assignee, Status, Test Cases, Time), the status transition workflow (New → ACK → Assigned → Resolved), and the requirement to document Root Cause Analysis (RCA) during resolution. It is part of the Data Quality and Observability section of the OpenMetadata documentation.
