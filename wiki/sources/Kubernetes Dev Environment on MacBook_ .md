# Kubernetes Dev Environment on MacBook

A comprehensive research report generated by Gemini on February 13, 2026, evaluating local Kubernetes development environments on macOS for Python-based geospatial ETL workloads. The report covers OrbStack and alternatives (Docker Desktop, Minikube, Kind, k3d, Rancher Desktop), general Kubernetes development best practices, and specific deployment patterns for the user's stack (Python/DuckDB/GeoPandas/GDAL for ETL, Airflow/Prefect for orchestration, FastAPI for APIs, with remote Oracle and PostGIS databases).

## Key Findings

- **OrbStack** is the recommended local Kubernetes solution for macOS due to exceptional performance, low resource consumption, and seamless Docker/K8s integration. Its paid commercial license is the main drawback.
- **Rancher Desktop** is the strongest free alternative, offering a similar integrated experience with k3s-based Kubernetes.
- **k3d** is best for CLI-focused, fast, efficient workflows without GUI requirements.
- **Docker Desktop** is resource-heavy and has licensing costs, making it less suitable.
- **Helm** significantly simplifies deploying stateful applications like Airflow and Prefect locally.
- **Kubernetes Jobs/CronJobs** are the natural workload type for ETL scripts, with proper resource requests/limits critical for geospatial processing.
- **Secrets management** is essential for external database connections — never hardcode credentials; use K8s Secrets mounted as volumes or env vars.
- **Multi-stage Docker builds** are crucial for GDAL/GeoPandas images to manage large dependencies while keeping final images small.
