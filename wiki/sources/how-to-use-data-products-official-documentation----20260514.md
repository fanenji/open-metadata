# How to Use Data Products | Official Documentation

This source is the official OpenMetadata v1.12.x documentation page for creating and managing Data Products. It provides a 5-step UI workflow: naming, describing, assigning ownership/experts, adding data assets, and viewing them on the Explore page. The document is purely procedural and lacks rationale, best practices, or technical details about the underlying metadata model.

## Key Content

- **Step 1**: Provide a Name (unique identifier) and Display Name (user-friendly label) for the Data Product.
- **Step 2**: Write a detailed Description explaining the purpose and contents of the Data Product.
- **Step 3**: Assign Owner/Experts (users or teams) for accountability and maintenance.
- **Step 4**: Add data assets (datasets, tables, dashboards) to the Data Product.
- **Step 5**: View and filter Data Product assets on the Explore page.

## Gaps

- No explanation of the relationship between Domains and Data Products.
- No API reference or JSON Schema details.
- No use cases, best practices, or troubleshooting guidance.
- No information on RBAC/ABAC interaction with Data Products.
