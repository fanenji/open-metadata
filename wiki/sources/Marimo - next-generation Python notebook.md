# Marimo - next-generation Python notebook

A marketing landing page for Marimo, an open-source, next-generation Python notebook. The page presents Marimo as a reactive, Git-friendly, and AI-native alternative to traditional Jupyter notebooks. It features a testimonial from Simon Willison highlighting the reactive execution model, and lists institutional users including Stanford, Johns Hopkins, UC Berkeley, Princeton, Columbia, SLAC, Mozilla AI, Chan Zuckerberg Initiative, OpenAI, BlackRock, Cloudflare, Pinterest, Sonos, and Owkin.

## Key Claims

- **Reactive:** Cells automatically re-execute when dependent values or code change.
- **Git-friendly:** Notebooks are stored as plain Python files, enabling standard version control workflows.
- **AI-native:** AI integration is a core design feature (claim is vague and unsubstantiated in the source).
- **Multi-output:** Can run as scripts, apps, or interactive notebooks.

## Evidence Strength

Weak. The source is a marketing page with no technical benchmarks, comparisons, or independent validation. The user list is a credibility signal but not evidence of technical superiority.

## Relevance to Data Platform

Marimo could serve as an exploratory analysis or prototyping tool within a data platform workflow, similar to Jupyter notebooks. The wiki currently has no coverage of notebook tools.
