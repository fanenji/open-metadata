# Create Tasks | OpenMetadata Collaboration Workflow

This source is the official overview page for the **Tasks** feature in OpenMetadata v1.12.x. It defines Tasks as an extension of [[conversation-threads]] that enables users to create structured requests for metadata changes — specifically, requests to create or update descriptions or tags on data assets.

The page establishes the default assignment logic: tasks are assigned to the data asset owner by default; if no owner exists, the task can be assigned to any appropriate user or team. This ties the task workflow directly to the [[data-asset-ownership]] model.

The source serves primarily as a navigation hub, linking to two sub-pages:
- **How to Request for Description** — detailed workflow for requesting description changes
- **How to Request for Tags** — detailed workflow for requesting tag changes (related to [[tag-request-workflow]])

The source is authoritative (official documentation) but brief, providing a high-level overview rather than detailed procedural or lifecycle information. It reinforces the [[bottom-up-top-down-enrichment]] concept by documenting a mechanism for top-down metadata enrichment through user-driven requests.
