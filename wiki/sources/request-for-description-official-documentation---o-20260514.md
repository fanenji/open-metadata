# Request for Description | Official Documentation - OpenMetadata Documentation

**Source:** https://docs.open-metadata.org/v1.12.x/how-to-guides/data-collaboration/request-description

Official documentation page (v1.12.x) describing two collaboration mechanisms for data asset descriptions: the Task-based description request workflow and threaded conversations around descriptions.

## Content Summary

The page covers two distinct features:

1. **How to Request for Description** — A formal workflow where users click the "?" icon next to a description field to create a Task. The Task is pre-populated with the title and shows Current, New, and Difference description fields. Users fill in assignees (users or teams) and the new description, then submit. Assignees can accept suggestions, edit and accept, add comments, or add other assignees. Created tasks appear in the Activity Feeds and Tasks tab for the data asset.

2. **Conversations around the Data Asset Description** — An informal mechanism where users click the Conversation icon next to a description to start a threaded discussion. Supports @mentions (tag users/teams), #mentions (tag data assets), replies, reactions, editing, and deletion.

The page explicitly links to the parallel [[how-to-request-for-tags]] workflow.

## Key Points

- Description requests create [[tasks]] in OpenMetadata
- Tasks are pre-populated with title and show Current/New/Difference views
- Assignees can accept, edit/accept, comment, or add other assignees
- Conversations around descriptions support @mentions, #mentions, replies, reactions, editing, and deletion
- Both mechanisms are part of the [[data-collaboration]] feature set
