# dbt Workflow | OpenMetadata Data Build Tool Integration

Official documentation for the dbt integration in OpenMetadata v1.12.x. Covers the requirements, artifact files (manifest.json, catalog.json, run_results.json), generation commands, storage configuration for dbt Core, dbt Cloud API integration, and the ten categories of metadata ingested from dbt.

## Key Content

- **Requirements**: Supports both dbt Core and dbt Cloud; dbt Core v1.2 through v1.9.
- **Artifact Files**: manifest.json (required), catalog.json (recommended), run_results.json (recommended).
- **Artifact Generation**: `dbt run`, `dbt test`, `dbt docs generate` sequence.
- **Storage**: For dbt Core, artifacts must be stored in S3, GCS, Azure, HTTP, or local filesystem.
- **dbt Cloud**: API-based integration; no external storage required.
- **Ten Metadata Categories**: Queries, Lineage, Tags, Owner, Descriptions, Tests/Test Results, Tiers, Glossary, Domain, Custom Properties.
- **Lineage Requirement**: `compiled_code` field must be present in manifest.json; generated by `dbt compile` and `dbt docs generate`.
- **Tests**: Only imported if run_results.json is provided.
