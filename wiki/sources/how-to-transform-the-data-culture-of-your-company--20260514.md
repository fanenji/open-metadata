# How to Transform the Data Culture of Your Company

This official OpenMetadata documentation page (v1.12.x) provides a strategic guide for transforming an organization's data culture using OpenMetadata. It defines data culture, explains its importance, outlines five key aspects, and details eight specific actions OpenMetadata enables to enhance data culture.

## Key Concepts

- **Data Culture**: A shared belief in using data to improve decision-making and performance, characterized by empowered people, data-prioritized decisions, and data-as-product management.
- **Data as a Product**: Treating data assets with documentation, SLAs, user understanding, continuous improvement, and feedback surveys.
- **Measure What Matters**: Setting KPIs to track ownership and documentation progress.
- **Reduce Toil**: Using integrated tools and automation over point solutions.
- **Organize for Data**: Decentralized, domain-based data ownership with dedicated resources.

## OpenMetadata Actions

1. Centralize metadata in OpenMetadata
2. Set KPIs to drive data ownership
3. Set KPIs to drive documentation
4. Develop data vocabulary using Glossary
5. Identify important data with Tiers (start with Tier 1 and Tier 5)
6. Provide feedback via weekly Data Insights reports
7. Use the Chrome Browser Extension for metadata consumption
8. Treat data as a product with profiling, quality tests, and collaboration
