# How Project Nessie Improves DataOps for the Lakehouse

This article, published by Dremio, argues that Project Nessie improves [[dataops-definition|DataOps]] by applying Git-like version control (branching, merging) to data lakehouses. It defines DataOps through four pillars (CI/CD, testing, orchestration, monitoring) and positions Nessie as a CI/CD enabler for data. Two illustrative scenarios are provided: a BI dashboard CSAT tracking use case and a data science social media sentiment integration use case, both demonstrating the branching workflow for safe, isolated experimentation. The article is a vendor-published thought piece and lacks quantitative evidence or comparison to alternatives.
