# Groovy to Python Migration

The process of refactoring legacy Groovy-based ETL and data management logic into a modern, maintainable Python stack. This migration focuses on:
- Changing configuration patterns (from `.properties` to `.env`).
- Improving CLI usability via `argparse`.
- Enhancing portability across Windows and Linux environments.
