# API Quality Criteria for HVD

Under the HVD regulation, all APIs serving High-Value Datasets must have published quality criteria covering:

- **Performance**: Response times and throughput
- **Capacity**: Maximum concurrent users and data volume
- **Availability**: Uptime guarantees and maintenance windows

For INSPIRE-compliant datasets, quality criteria follow EU Regulation 976/2009 (as amended by 1088/2010). For non-INSPIRE datasets, the data owner defines and publishes these criteria. API documentation must be in OpenAPI 3.0 (REST) or WSDL 1.1 (SOAP) format.
