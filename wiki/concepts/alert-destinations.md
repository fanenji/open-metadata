# Alert Destinations

Alert Destinations are configurable targets for delivering notifications generated by OpenMetadata's alerting system. They are the output side of the [[data-observability-alerts]] and [[system-governance-notifications]] framework, determining where and how alert messages are received.

## Supported Destinations

OpenMetadata supports the following alert destinations:

- **Slack** — via Incoming Webhooks (see [[slack-alerts-configuration]])
- **Email** — SMTP-based email delivery
- **Microsoft Teams** — via Incoming Webhooks
- **Google Chat** — via Incoming Webhooks
- **Generic Webhook** — custom HTTP endpoint for any webhook-compatible service

## Common Configuration Pattern

All webhook-based destinations follow a similar two-phase setup:

1. **External setup**: Create a webhook URL in the target platform (Slack, Teams, Google Chat, etc.).
2. **OpenMetadata setup**: Add the destination in Alerts & Notifications, paste the webhook URL, test the connection, and save.

## Best Practices

- Use descriptive display names for each destination to distinguish alert types.
- Test connections after initial setup and after any webhook URL changes.
- Consider using multiple destinations to route different alert categories to different channels or teams.
- Monitor destination health; webhook URLs can expire or be revoked.
