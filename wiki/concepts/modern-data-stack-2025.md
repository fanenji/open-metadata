# The Modern Data Stack in 2025: What Actually Won

This synthesis page summarizes the key findings from the analysis of the modern data stack in 2025.

## Winners by Category

| Category | Winner | Adoption | Trend |
|----------|--------|----------|-------|
| Warehouse | Snowflake | 41% | Stable |
| Warehouse (fastest growing) | Databricks | 32% | +60% YoY |
| Transformation | dbt | 73% | Near-monopoly |
| Ingestion | Fivetran | 34% | Leader |
| Ingestion (fast follower) | Airbyte | 21% | +91% YoY |
| BI (enterprise) | Tableau | 38% | Stable |
| BI (growing) | Power BI | 33% | Growing |
| Orchestration | Fragmented | — | No clear winner |
| Observability | DIY (dbt tests + custom) | 42% | Dominant |

## Key Themes

1. **[[multi-warehouse-reality|Multi-warehouse is the new normal]]** — 38% of companies use 2+ warehouses.
2. **[[platformization|Platformization]]** — Warehouse vendors becoming full platforms.
3. **[[diy-persistence|DIY persistence]]** — Custom solutions remain common (19–42% depending on category).
4. **[[sql-dominance|SQL dominance]]** — 94% of transformation logic is SQL.
5. **[[reverse-etl-consolidation|Reverse ETL consolidation]]** — Predicted to merge into ingestion tools.
6. **[[failed-predictions-2020|Failed predictions]]** — Data mesh, real-time, DataOps did not materialize.

## Implications for Practitioners

Focus skills for 2025–2027:
1. SQL (still foundation)
2. dbt (industry standard)
3. One major warehouse deeply (Snowflake or Databricks)
4. Python (for ML and custom logic)
5. Basic orchestration (tool-agnostic concepts)
