# dbt Labs

dbt Labs is the organization behind [[dbt Core]] and [[dbt Cloud]], the industry-standard transformation tools for analytics engineering. Founded in May 2016 under the name [[Fishtown Analytics]], the company later rebranded to dbt Labs. They created and popularized the [[analytics-engineering-role|analytics engineering]] role and drive the development of official adapters for major cloud data warehouses, including native support for Python models in Snowflake, Databricks, and BigQuery. dbt Labs also hosts the annual [[coalesce-conference|Coalesce]] conference, featuring dbt users and use cases from the community.

In addition to their core data transformation tools, dbt Labs is a thought leader in data mesh. They published "The 4 principles of data mesh," which defines the core principles of [[data-mesh]]: domain ownership, data as a product, self-serve data platform, and federated computational governance. Their products [[dbt Cloud]] and [[dbt-mesh]] are referenced in the [[data-mesh-maturity-assessment]] as indicators of maturity in the "data as a product" dimension.

## Key Products

- **[[dbt-core|dbt Core]]** — Open-source transformation tool that enables analytics engineers to transform data in their warehouse. dbt Labs maintains official adapters for major cloud data warehouses.
- **[[dbt-cloud|dbt Cloud]]** — Managed platform for deploying, orchestrating, and monitoring dbt projects.
- **[[dbt-mesh]]** — Enables data mesh architectures within dbt Cloud for domain ownership and data as a product.
- **[[dbt-fusion]]** — Advanced SQL compilation engine with column-level lineage.
- **[[dbt-semantic-layer]]** — Governed metrics interface for consistent metric definitions.
- **[[dbt-mcp-server]]** — MCP server for AI agent interaction with dbt projects.
- **dbt Copilot** — AI assistant for dbt (credit-based consumption).
