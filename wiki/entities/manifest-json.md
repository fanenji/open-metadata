# manifest.json

`manifest.json` is a core artifact generated by dbt (Data Build Tool). It contains a complete representation of a dbt project, including models, sources, tests, and their dependencies. OpenMetadata uses this file as the primary source for ingesting dbt lineage.

## Key Fields for Lineage

- **`depends_on`** — Lists upstream node dependencies (macros and nodes). OpenMetadata reads the `nodes` array to create direct lineage edges.
- **`compiled_code` / `compiled_sql`** — Contains the compiled SQL query for each model. OpenMetadata's Lineage parser uses this to extract source and target tables.

## Prerequisites

To ensure `compiled_code` is populated, run:
```
dbt compile
dbt docs generate
```

If `compiled_code` is missing for a node, lineage will not be captured for that node.
